## 2024-05-18 - Nested Interactive Elements in Custom Accordions
**Learning:** In the `NfseList.jsx` component, the main `div` elements used as accordion headers ("Company Header" and "Competence Sub-Header") also contained nested interactive `<button>` elements (e.g., for opening folders or downloading ZIPs). Converting the outer wrapper to a native `<button>` element to gain free keyboard accessibility would result in invalid HTML (`<button>` inside `<button>`), breaking browser parsing and screen reader experience.
**Action:** When improving accessibility for interactive list or accordion headers that contain nested action buttons, you must retain the outer `<div>` and manually construct the accessible button semantics by adding `role="button"`, `tabIndex={0}`, `aria-expanded`, and manual `onKeyDown` handlers for `Enter` and `Space`. Remember to call `e.preventDefault()` on the Spacebar event to prevent unintended page scrolling.

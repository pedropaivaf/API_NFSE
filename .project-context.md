## O que é este projeto

Aplicação **desktop Electron** para empresas brasileiras baixarem automaticamente XMLs de **Notas Fiscais de Serviço (NFS-e)** do Portal Nacional do Governo Federal (nfse.gov.br).

Autenticação via **mTLS com Certificado Digital A1 (.pfx)** ou **usuário/senha**. Os XMLs são persistidos localmente e os metadados no **Supabase (PostgreSQL)**.

## Fluxo Principal (mTLS)

```
1. Usuário seleciona .pfx + senha
2. POST /scraper/validate-cert → valida cert, identifica empresa pelo CNPJ
3. Usuário clica "Iniciar Extração"
4. POST /scraper/fetch-gov:
   a. nfseScraperService autentica: GET /EmissorNacional/Certificado (TLSv1.2 + pfx)
   b. 302 → /Dashboard → extrai JWT (window.sessionStorage)
   c. GET /Notas/Recebidas?dataInicio=DD/MM/YYYY&dataFim=DD/MM/YYYY
      Authorization: Bearer {JWT}
   d. Parse HTML/JSON com Cheerio
   e. Upsert metadados → Supabase tabela `nfs`
   f. Download XMLs → ~/Documents/notas_processadas/{tipo}/
```

## Convenções Importantes

- **Período máximo de consulta**: 30 dias (limite do portal NFSe Gov.br). A opção "Mês Atual" sempre usa os últimos 30 dias; "Personalizado" é limitado a 30 dias de intervalo.
- **TLS**: forçar `maxVersion: 'TLSv1.2'` — IIS 10.0 do gov.br tem bugs com client cert em TLS 1.3
- **Senhas de certificado**: salvas em texto plano no banco (MVP — criptografar em produção)
- **Chave service_role**: usada apenas no backend, nunca exposta no frontend
- **Documentação**: atualizar o doc correspondente em `docs/` antes de cada commit/push

## Fluxo Principal (ExtraГ§ГЈo RPA via Scraper)

```
1. UsuГЎrio seleciona .pfx + senha  (ou usuГЎrio/senha)
2. POST /scraper/validate-cert вҶ’ valida cert, identifica empresa pelo CNPJ
3. UsuГЎrio clica "Iniciar ExtraГ§ГЈo"
4. POST /scraper/fetch-gov:
   a. nfseScraperService autentica: GET /EmissorNacional/Certificado (mTLS TLSv1.2)
   b. 302 вҶ’ /Dashboard вҶ’ extrai JWT (window.sessionStorage)
   c. GET /EmissorNacional/Notas/Recebidas?dataInicio=DD/MM/YYYY&dataFim=DD/MM/YYYY
      Authorization: Bearer {JWT}
   d. Parse HTML/JSON com Cheerio
   e. Upsert metadados вҶ’ Supabase tabela `nfs`
   f. Download XMLs вҶ’ {output_path}/recebidas/ ou {output_path}/emitidas/
      (padrГЈo: ~/Documents/notas_processadas/)
```

## Fluxo Alternativo (Sync via API gov.br)

```
POST /companies/:id/sync
вҶ’ nfseService.syncNfse(companyId)
вҶ’ Baixa certificado do Supabase Storage
вҶ’ Autentica com mTLS na API REST do gov.br
вҶ’ Para cada nota encontrada:

```
